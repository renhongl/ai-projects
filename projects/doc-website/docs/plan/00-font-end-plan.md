# 前端学习文件夹

## 背景
- 你是前端开发专家
- 精通前端开发知识，包括但不限于：前端基础、react、nodejs、agent开发、前端各种前沿技术
- 精通各种复杂系统的设计、开发
- 精通前端开发专家需要掌握的知识


## 前端学习规划
目标：在 6 个月内建立从前端基础、工程化、框架原理、Node.js、系统设计到 AI Agent 应用开发的完整能力体系，达到可以独立负责复杂前端项目设计、开发、优化和落地的水平。

### 第一阶段：夯实前端基础（第 1-4 周）
- 深入掌握 HTML、CSS、JavaScript、TypeScript 的核心语法与底层原理。
- 系统理解浏览器工作机制，包括渲染流程、事件循环、网络请求、缓存策略和跨域机制。
- 补齐 Web 标准与协议基础，包括 HTTP/HTTPS、DNS、CDN、Cookie、存储、浏览器安全模型。
- 学习模块化、函数式编程、面向对象设计、设计模式等基础能力。
- 输出要求：
  - 手写常见 JavaScript 核心能力，如 `call`、`apply`、`bind`、`Promise`、防抖、节流、深拷贝。
  - 完成一个不依赖框架的中小型页面项目。
  - 整理一份前端基础知识笔记。

### 第二阶段：掌握 React 与现代前端开发（第 5-8 周）
- 深入学习 React 核心概念：组件化、状态管理、Hooks、生命周期、渲染机制。
- 理解 React 18 常用能力与现代开发模式，包括并发更新、状态拆分、组件复用。
- 学习前端路由、表单管理、数据请求、错误处理、权限控制等业务开发能力。
- 学习可访问性设计与实现，包括语义化标签、键盘可用性、屏幕阅读器兼容、表单无障碍。
- 输出要求：
  - 从零实现一个 React 中后台项目。
  - 封装一套可复用的业务组件。
  - 总结 React 性能优化方法与常见陷阱。

### 第三阶段：工程化与质量保障（第 9-12 周）
- 学习 Vite、Webpack 基础能力，理解构建、打包、代码分割、按需加载、资源优化。
- 掌握 ESLint、Prettier、Husky、Lint-Staged、Commit 规范、CI/CD 基本流程。
- 学习测试体系，包括单元测试、集成测试、端到端测试。
- 学习前端发布策略与环境治理，包括多环境配置、灰度发布、版本回滚、Feature Flag。
- 输出要求：
  - 为已有项目搭建完整工程化方案。
  - 补齐测试用例和基础自动化流程。
  - 写一份“项目工程规范文档”。

### 第四阶段：深入 Node.js 与全栈协作（第 13-16 周）
- 学习 Node.js 基础、模块系统、异步模型、文件系统、网络编程。
- 掌握 Express、NestJS 或同类服务端框架的基本使用。
- 学习数据库基础、鉴权、日志、接口设计、部署流程。
- 学习常见服务端安全问题，包括鉴权鉴别、XSS、CSRF、注入、限流、敏感信息保护。
- 输出要求：
  - 开发一个前后端联调项目，包含登录、权限、列表、详情、上传等常见能力。
  - 独立设计一套 RESTful API 或 BFF 服务。
  - 整理前后端协作规范。

### 第五阶段：复杂系统设计与性能优化（第 17-20 周）
- 学习大型前端项目的目录设计、模块拆分、状态架构、组件分层和可维护性设计。
- 掌握性能优化方法：首屏优化、懒加载、缓存、SSR/SSG、监控与埋点。
- 学习可观测性建设，包括错误监控、性能指标、日志追踪。
- 学习稳定性治理，包括降级、熔断、重试、兜底、异常恢复、线上问题排查。
- 输出要求：
  - 对一个已有项目做系统性性能分析与优化。
  - 产出一份架构设计文档和优化报告。
  - 建立一套可复用的监控与告警方案。

### 第六阶段：前沿方向与 Agent 开发（第 21-24 周）
- 学习 AI 应用开发基础，理解 Prompt、工具调用、工作流编排、RAG、Agent 基本模式。
- 掌握前端与 AI 结合的常见场景，如智能问答、自动化助手、内容生成、数据分析增强。
- 学习 MCP、插件机制、命令行 Agent、Web Agent 等相关实践。
- 输出要求：
  - 开发一个实际可用的前端 AI Agent 项目。
  - 封装一个带工具调用能力的前端应用或插件。
  - 总结 AI Agent 在前端领域的应用场景与限制。

### 第七阶段：补齐前端专家专项能力（第 25-28 周）
- 学习跨端与多端能力，包括移动端适配、响应式设计、小程序、Hybrid、Electron 或 React Native 基础认知。
- 学习设计系统与组件库建设，包括 Token、主题系统、组件规范、文档站点、版本管理。
- 学习国际化与本地化，包括文案管理、日期时间、货币格式、RTL 兼容。
- 学习数据可视化与交互设计基础，包括图表选型、复杂表格、动画反馈和信息层级表达。
- 学习团队协作能力，包括需求拆解、技术方案评审、Code Review、研发流程推动和跨团队协作。
- 输出要求：
  - 搭建一套基础组件库或设计系统 Demo。
  - 完成一个移动端或跨端项目原型。
  - 输出一份团队级前端协作规范与评审清单。

### 持续性要求
- 每周至少完成 1 个知识专题总结。
- 每两周完成 1 个小项目或核心模块练习。
- 每月完成 1 个可展示的完整项目。
- 持续维护知识库，沉淀代码片段、设计方案、问题复盘和性能优化经验。
- 每个季度至少完成 1 次系统化复盘，更新自己的能力矩阵和短板清单。

### 前端专家补充能力清单
- 安全能力：理解 XSS、CSRF、点击劫持、依赖投毒、权限边界和安全治理流程。
- 可访问性能力：能让核心页面满足基础无障碍要求，而不是只追求“能显示”。
- 跨端能力：能处理 PC、移动端、H5、小程序、桌面端等多终端差异。
- 运维与交付能力：理解部署、监控、灰度、回滚和线上问题止损机制。
- 协作与治理能力：能推动规范、评审、复盘和团队工程质量提升。
- 业务抽象能力：能把复杂业务沉淀为组件、Hooks、SDK、脚手架或平台能力。
- 技术决策能力：能在性能、成本、复杂度、可维护性之间做清晰取舍。

### 建议学习方法
- 采用“学知识 + 做项目 + 写总结 + 做复盘”的闭环。
- 每个阶段都必须有可运行项目，而不是只看资料。
- 优先理解原理，再追求工具熟练度。
- 对照真实业务场景训练架构设计、代码质量和协作能力。
- 不只学习“怎么做”，还要学习“为什么这样做”和“不这样做会有什么代价”。

### 最终目标
- 能独立负责复杂前端项目从方案设计到落地交付。
- 能承担前端架构、工程化建设、性能优化和稳定性治理工作。
- 能完成 Node.js 服务协作与 BFF 设计。
- 能探索并落地 AI Agent 与前端结合的实际项目。
- 能在团队内承担技术负责人角色，推动规范、质量和平台能力建设。

## 版本 B：前端专家能力矩阵与自测清单

使用方式：
- 每个能力项按 `1-4` 自评。
- `1` 代表了解概念。
- `2` 代表能跟着资料完成。
- `3` 代表能独立落地并解决常见问题。
- `4` 代表能制定规范、指导他人、完成复杂场景决策。

### 1. 前端基础能力
- HTML 语义化、表单、媒体、SEO 基础是否扎实。
- CSS 布局、响应式、动画、层叠、BFC、包含块、定位机制是否理解。
- JavaScript 原型链、闭包、作用域、异步模型、事件循环是否能清楚解释。
- TypeScript 泛型、类型推导、条件类型、映射类型、类型体操基础是否掌握。
- 浏览器渲染流程、重排重绘、缓存、存储、跨域是否理解。
- 自测问题：
  - 能否独立说明从输入 URL 到页面渲染完成的全过程？
  - 能否手写 Promise、发布订阅、防抖节流、深拷贝等常见基础实现？
  - 能否解释为什么某段 CSS 不生效，而不是靠试错？

### 2. Web 标准与网络协议能力
- HTTP/HTTPS、TLS、DNS、CDN、Cookie、Session、LocalStorage、IndexedDB 是否理解。
- 浏览器安全模型、同源策略、CORS、缓存协商、内容协商是否理解。
- 自测问题：
  - 能否解释强缓存与协商缓存的区别和适用场景？
  - 能否分析一次接口慢请求是前端、网络还是服务端问题？
  - 能否设计静态资源缓存与更新策略？

### 3. React 与现代框架能力
- React 组件设计、Hooks、Context、状态边界、渲染机制是否掌握。
- React 性能优化、Suspense、并发更新、错误边界、代码拆分是否掌握。
- 状态管理、路由管理、表单管理、请求层抽象、权限系统是否能独立设计。
- 自测问题：
  - 能否说清一个组件为什么重复渲染？
  - 能否给一个中后台项目设计清晰的状态分层？
  - 能否把一个复杂业务模块拆成可维护的组件和 Hooks？

### 4. 工程化与研发效能能力
- Vite、Webpack、Babel、ESBuild、代码分包、按需加载、Tree Shaking 是否理解。
- ESLint、Prettier、Commit 规范、脚手架、Monorepo、CI/CD 是否熟悉。
- 测试体系是否覆盖单元测试、集成测试、端到端测试。
- 自测问题：
  - 能否从零搭建一个可交付的前端工程模板？
  - 能否解释一次构建过慢的原因并提出优化方案？
  - 能否设计多人协作下稳定可执行的研发流程？

### 5. 架构设计与业务抽象能力
- 是否能为大型项目设计目录结构、模块边界、组件层次、状态架构。
- 是否能把业务沉淀为组件、Hooks、SDK、脚手架、平台能力。
- 是否能在复用性、复杂度、研发效率之间做取舍。
- 自测问题：
  - 能否为一个复杂项目写出架构设计文档？
  - 能否判断哪些逻辑应放业务层，哪些应沉淀为公共层？
  - 能否解释某个技术方案的收益、成本和替代方案？

### 6. 性能优化与稳定性能力
- 首屏优化、懒加载、图片优化、缓存、SSR/SSG、长列表优化是否掌握。
- 错误监控、性能监控、埋点、日志追踪、告警机制是否掌握。
- 降级、熔断、重试、兜底、异常恢复、线上排障是否具备实战认知。

  - 能否基于指标定位首屏慢的主要瓶颈？

## 版本 A：前 4 周详细执行计划

执行原则：
- 每周覆盖 `学习输入 + 编码练习 + 输出总结 + 自测复盘` 四部分。
- 每周至少保证 5 天投入，每天 1.5-3 小时。
- 每周末必须产出可以看得见的结果，而不是只有阅读记录。

### 第 1 周：JavaScript 与浏览器基础校准
- 学习重点：
  - 作用域、闭包、this、原型链、继承、执行上下文。
  - 事件循环、宏任务微任务、Promise、async/await。
  - 浏览器渲染流程、重排重绘、缓存基础。
- 编码练习：
  - 手写 `call`、`apply`、`bind`、`new`、`Promise` 简化版。
  - 实现防抖、节流、深拷贝、发布订阅。
- 输出物：
  - 一份“JavaScript 核心机制”总结。
  - 一个 `js-practice` 代码目录，包含上述实现与注释。
- 验收标准：
  - 能不看资料解释事件循环。
  - 手写题能在 30-45 分钟内完成主要版本。

### 第 2 周：HTML/CSS 与页面还原能力
- 学习重点：
  - 语义化、表单、可访问性基础。
  - 盒模型、BFC、Flex、Grid、定位、层叠上下文、响应式设计。
  - 常见页面布局与移动端适配。
- 编码练习：
  - 独立还原一个中等复杂度静态页面，包含导航、卡片区、表单区和响应式布局。
  - 为表单补齐标签关联、键盘可用性和错误提示。
- 输出物：
  - 一个纯 HTML/CSS/JS 页面项目。
  - 一份“CSS 布局与定位问题排查清单”。
- 验收标准：
  - 页面在桌面端和移动端都可用。
  - 能解释布局方案为什么这样选，而不是只追求“看起来对”。

### 第 3 周：TypeScript 与 Web 协议基础
- 学习重点：
  - TypeScript 类型系统、泛型、联合类型、交叉类型、工具类型。
  - HTTP/HTTPS、DNS、CDN、Cookie、LocalStorage、SessionStorage、CORS。
  - 浏览器安全模型与基础安全认知。
- 编码练习：
  - 用 TypeScript 重写第 1 周的核心手写题。
  - 实现一个带类型约束的请求封装模块。
- 输出物：
  - 一份“TypeScript 常用模式”总结。
  - 一份“Web 协议与安全基础”笔记。
- 验收标准：
  - 能为常见业务数据结构写出清晰类型定义。
  - 能解释跨域、缓存和 Cookie/Token 的基本边界。

### 第 4 周：综合小项目与阶段复盘
- 学习重点：
  - 把前 3 周内容整合到真实项目里。
  - 训练模块拆分、基础工程组织和问题复盘能力。
- 编码练习：
