# 代码分割与按需加载

## 这是什么
代码分割是把大包拆成多个更小的包，按需加载是在真正需要时再加载对应资源。

## 为什么重要
- 降低首屏体积。
- 提升加载速度。
- 降低用户首次访问成本。

## 重点理解
- 路由级分包。
- 组件级懒加载。
- 动态导入。
- 公共包拆分策略。

## 你需要掌握的能力
- 在 React 项目中实现懒加载。
- 理解公共依赖为什么会被抽成独立 chunk。
- 能分析分包是否真的有效。

## 实践任务
- 给一个项目接入动态 `import()`。
- 对某个页面做路由级懒加载。
- 观察构建产物中 chunk 的变化。

## 自测清单
- 我能说清为什么不是拆得越碎越好吗？
- 我能解释懒加载对首屏和交互的影响吗？
- 我能判断哪些模块适合独立拆包吗？
