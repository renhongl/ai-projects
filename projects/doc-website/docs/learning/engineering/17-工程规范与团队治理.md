# 工程规范与团队治理

## 这是什么
这部分关注的不是单个工具，而是如何把工具、流程和规则真正变成团队稳定能力。

## 核心内容
- 目录结构规范
- 命名规范
- 组件设计规范
- API 请求层规范
- 状态管理规范
- 错误处理规范
- 发布与复盘规范

## 为什么重要
- 工具只能解决一部分问题。
- 真正决定团队质量的，是一致的工程约束和持续执行。

## 你需要掌握的能力
- 能输出项目工程规范文档。
- 能推动规范落地，而不是只写文档。
- 能在效率和约束之间做平衡。

## 实践任务
- 写一份适合个人项目的最小工程规范。
- 设计一份 Code Review 清单。
- 记录三类常见协作问题及治理方式。

## 自测清单
- 我能判断哪些规范值得强制，哪些只需要建议吗？
- 我能通过规范降低沟通成本，而不是增加流程负担吗？
- 我能把个人习惯沉淀成团队可复用能力吗？
