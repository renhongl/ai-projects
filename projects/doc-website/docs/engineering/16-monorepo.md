# Monorepo 与脚手架

## 这是什么
`Monorepo` 是把多个相关项目放在一个仓库中统一管理，脚手架用于标准化项目初始化过程。

## 为什么重要
- 多项目共享代码和规范更方便。
- 统一依赖和工具链更容易治理。
- 脚手架可以降低重复搭建成本。

## 重点理解
- 什么场景适合 Monorepo。
- 包管理、共享配置、版本管理的基本思路。
- 脚手架如何沉淀团队最佳实践。

## 你需要掌握的能力
- 了解 Monorepo 基础结构。
- 能抽出可复用配置。
- 能设计一个简单的项目初始化模板。

## 实践任务
- 研究一个开源 Monorepo 项目结构。
- 设计一个适合自己学习项目的初始化模板。
- 总结 Monorepo 的优势与成本。

- 我能说清脚手架要解决的核心问题吗？
