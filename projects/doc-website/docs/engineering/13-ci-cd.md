# CI/CD

## 这是什么
`CI/CD` 指持续集成和持续交付，用自动化流程把代码检查、测试、构建、发布串起来。

## 核心价值
- 降低人工操作风险。
- 保证每次变更都有统一检查。
- 提高发布稳定性和频率。

## 典型流程
- 安装依赖
- 运行 lint
- 运行测试
- 构建产物
- 发布或部署

## 你需要掌握的能力
- 理解持续集成的基本链路。
- 设计最小 CI 流程。
- 识别哪些步骤必须阻塞，哪些可以异步。

## 实践任务
- 为项目设计一条基础 CI 流程。
- 规定主分支合入前必须通过的检查。
- 记录失败构建的常见原因。

## 自测清单
- 我能解释为什么 CI 是团队下限保障吗？
- 我能设计一条不过重但有效的流水线吗？
- 我能区分“本地检查”和“流水线检查”的边界吗？
