# 工程化总览

## 这是什么
前端工程化的核心目标，是把“能跑的代码”变成“可协作、可维护、可发布、可持续演进的项目”。

## 主要能力模块
- 构建能力：`Vite`、`Webpack`、`Babel`、`ESBuild`
- 打包优化：代码分割、按需加载、缓存、压缩、资源治理
- 代码规范：`ESLint`、`Prettier`
- 提交流程：`Husky`、`lint-staged`、Commit 规范
- 测试体系：单元测试、集成测试、端到端测试
- 交付能力：`CI/CD`、多环境配置、灰度发布、回滚
- 仓库治理：`Monorepo`、脚手架、统一规范

## 为什么重要
- 没有工程化，项目容易失控。
- 没有规范，团队协作成本会持续上升。
- 没有发布和回滚能力，线上风险不可控。
- 没有自动化，研发效率无法稳定提升。

## 学习顺序
1. 先理解构建和打包。
2. 再建立代码规范和提交流程。
3. 再补测试与持续集成。
4. 最后学习发布治理和团队级工程体系。

## 实践目标
- 能从零搭一个前端项目模板。
- 能解释构建慢、包大、规范混乱、发布不稳时该怎么处理。
- 能推动团队形成稳定的研发流程。
