# Prettier

## 这是什么
`Prettier` 是代码格式化工具，目标是统一代码排版，减少无意义争论。

## 核心价值
- 自动统一代码样式。
- 降低格式类 Code Review 成本。
- 与 `ESLint` 配合提升一致性。

## 重点理解
- `Prettier` 管排版，`ESLint` 更偏质量检查。
- 两者冲突时要做好规则边界管理。

## 你需要掌握的能力
- 为项目接入 `Prettier`。
- 配置编辑器自动格式化。
- 处理与 `ESLint` 的冲突。

## 实践任务
- 创建项目格式化配置。
- 配置保存时自动格式化。
- 确保提交前代码风格统一。

## 自测清单
- 我能说清 `Prettier` 和 `ESLint` 的职责边界吗？
- 我能处理常见的格式化冲突吗？
- 我知道哪些场景不该过度定制格式规则吗？
