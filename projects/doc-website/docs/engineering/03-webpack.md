# Webpack

## 这是什么
`Webpack` 是经典的前端模块打包工具，灵活性高，适合理解构建系统底层机制。

## 核心价值
- 把各种模块、样式、资源打包成浏览器可运行的产物。
- 拥有强大的 Loader 和 Plugin 生态。
- 能高度定制大型项目构建流程。

## 重点理解
- Entry、Output、Loader、Plugin 的职责。
- 模块依赖图是如何建立的。
- 开发模式与生产模式的区别。
- 常见优化项：分包、缓存、压缩、Source Map。

## 你需要掌握的能力
- 能看懂基础 `webpack.config.js`。
- 能为 CSS、图片、TypeScript、JSX 配置处理链路。
- 能理解常见插件的作用。
- 能定位基本构建问题。

## 实践任务
- 手写一个最小可运行的 Webpack 项目配置。
- 为项目补齐 `css`、图片和 TypeScript 支持。
- 开启代码分割并对比打包前后产物。

## 自测清单
- 我能说清 Loader 和 Plugin 的区别吗？
- 我能描述从入口文件到最终 bundle 的处理流程吗？
- 我能定位一次打包慢的大致原因吗？
