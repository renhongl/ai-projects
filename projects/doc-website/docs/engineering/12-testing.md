# 测试体系

## 这是什么
测试体系是保证代码行为稳定的重要手段，常见分为单元测试、集成测试、端到端测试。

## 三类测试
- 单元测试：验证最小功能单元。
- 集成测试：验证模块之间组合行为。
- 端到端测试：验证用户真实操作路径。

## 为什么重要
- 防止回归。
- 提升重构信心。
- 为持续交付提供质量基础。

## 你需要掌握的能力
- 能根据场景选择测试层级。
- 为核心逻辑写单元测试。
- 为关键流程设计 E2E 测试。

## 实践任务
- 为一个工具函数写单元测试。
- 为一个表单提交流程写集成测试。
- 为一个登录流程写 E2E 用例。

## 自测清单
- 我知道哪些逻辑值得测，哪些不值得过度测试吗？
- 我能解释为什么测试金字塔仍然有参考价值吗？
- 我能设计一套最小但有效的测试策略吗？
