# 资源优化与 Tree Shaking

## 这是什么
资源优化是减少不必要的体积和请求成本，`Tree Shaking` 是在打包阶段删除未使用代码的一种方式。

## 重点内容
- JS 压缩与去死代码。
- CSS 压缩与提取。
- 图片压缩与格式选择。
- 字体、图标、静态资源缓存策略。
- `Tree Shaking` 的原理与限制。

## 为什么重要
- 包体积直接影响加载速度。
- 资源治理影响用户体验和服务器成本。

## 你需要掌握的能力
- 知道如何分析包体积。
- 理解为什么某些代码无法被 `Tree Shaking`。
- 能制定静态资源优化方案。

## 实践任务
- 使用打包分析工具查看产物体积。
- 找出一个项目中可优化的依赖或资源。
- 对比优化前后的构建体积与加载结果。

## 自测清单
- 我能解释 `Tree Shaking` 为什么依赖静态模块结构吗？
- 我知道图片、字体、图标分别该如何优化吗？
- 我能判断是“代码问题”还是“依赖引入方式”导致包大吗？
